01  Project 실무 · 최신

SaaS 멀티테넌트 LMS 플랫폼

메가존클라우드 인턴십 · 2025.11 – 2026.01 · 팀 7명

하나의 플랫폼에서 여러 기업이 독립적으로 서비스를 사용할 수 있도록 멀티테넌트 구조를 설계한 SaaS LMS입니다. 데이터 격리 · 도메인 무결성 · 동시성 · AI 협업 네 축을 설계하고 검증했습니다.

▸

주의가 아니라 구조로. 개발자가 tenant_id를 빠뜨려도, 상태를 잘못 바꾸려 해도, 동시 요청이 몰려도 구조가 먼저 막도록 설계했습니다. AI 협업까지 같은 원리로 — 개인의 프롬프트 역량이 아니라 팀의 표준 컨텍스트가 품질을 보장하게 했습니다.

Work

멀티테넌트 데이터 격리 3단계 방어 구조 설계 — TenantResolver Filter가 서브도메인에서 tenant_id를 추출해 ThreadLocal에 저장, Security Filter가 JWT 클레임과 대조해 불일치 시 403, Hibernate Filter가 모든 Repository Query에 조건을 자동 주입
도메인 라이프사이클 교육 운영 핵심 도메인(Time Schedule) 전담 — 요구사항 분석부터 데이터 모델링, REST API까지. 상태 전이를 엔티티 도메인 메서드로 캡슐화(DRAFT → RECRUITING → ONGOING → CLOSED → ARCHIVED)하고, ValidationRule enum으로 커스텀 제약 19종 관리
성능 · 동시성 고충돌 도메인에 비관적 락 4개, 저충돌 도메인에 @Version 낙관적 락 15개로 락 전략 분리. Projection 8종 · GROUP BY 집계 47개 · Fetch Join 18개로 N+1 차단. 575개 테스트와 멀티스레드 동시성 테스트 2종으로 검증
AI 협업 체계 CLAUDE.md 기반 Context Engineering 표준 설계 — 계층 책임 · DTO/Entity 규칙 · 예외 포맷 · Git/PR 컨벤션 · 프롬프트 템플릿 · AI 작업 절차를 문서화해 7명이 같은 기준으로 LLM 활용

02  Project ★ 최우수상

SaaS 기반 클라우드 ERP

메가존클라우드 SaaS Java 개발자 양성과정 · 2024.05 – 2024.12 · 팀 7명

SaaS 기반 클라우드 ERP에서 물류 도메인 개발과 오픈소스 모니터링 구축, Redis 캐싱 전략을 담당했습니다. 3개 팀 중 1위로 최우수상을 수상했습니다.

▸

Observe → Validate → Improve. 모니터링으로 운영 데이터를 확보하고, K6 부하 테스트로 서비스 경계를 검증한 뒤, 정합성 우선 Redis 캐싱으로 DB 병목을 해소했습니다.

Work

오픈소스 모니터링 Helm으로 Prometheus·Grafana 구축, kube-state-metrics와 Node Exporter로 클러스터 내부 지표 수집, CloudWatch Exporter를 추가해 RDS·ALB까지 하나의 대시보드에서 통합 관측 — values.yaml에 필요한 Namespace·Dimension만 직접 매핑해 핵심 지표만 스크랩
MSA 경계 검증 K6(200 VU) 부하 테스트로 응답 시간 · 처리량 · 자원 사용량을 함께 분석해, 설계한 도메인 경계가 실제 트래픽 패턴과 맞는지 검증 (서비스 물리 분리는 팀 공동 수행)
Redis 캐싱 Read-Through / Write-Through 전략. RedisTemplate과 AOP로 커스텀 어노테이션(@RedisCacheable, @RedisCachePut)을 만들어 메서드 단위 동적 TTL 제어, JPA Metamodel로 엔티티 연관관계를 런타임 분석해 정·역방향 Dependency Graph 자동 구성
